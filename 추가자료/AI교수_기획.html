§1

프로젝트 개요

한 줄 정의 — 슬라이드 기반으로 살아있는 강의를 하고, 오프라인 시험을 출제·채점하는 AI 교수.

과목프로그래밍 언어론 (Programming Languages / 언어 생성 규칙)
수준대학 3학년 전공
형태여러 학생 대상 강의
AI 역할가르치기 + 시험 출제 + 채점

다루는 주제 — 형식 문법(BNF/EBNF, CFG), 파스 트리, 구문 분석, 의미론, 타입 시스템, 계산 모델(람다 계산법·패러다임), 언어 설계.

§2

핵심 설계 원칙

성격이 정반대인 두 세계로 시스템을 나누는 것이 아키텍처의 근간이다. 배움은 따뜻하고 개인화되며, 평가는 건조하고 획일적이다.

🎭 배움의 세계 persona: ON

  • 슬라이드 강의
  • QnA 튜터
  • 학생이 교수 스타일 선택

친근함 · 개인화 — 학생마다 다른 교수, 다른 말투

약점 데이터 → 맞춤 출제

⚖️ 평가의 세계 persona: OFF

  • 오프라인 시험 출제
  • 채점 (자동검증 + 루브릭)
  • 모두에게 동일 기준

일관성 · 정당성 — 모두에게 똑같은 잣대

데이터는 한 방향으로만 흐른다. 배움이 쌓은 약점 데이터는 맞춤 출제에 쓰이되, 채점이 페르소나의 영향을 받는 역방향은 원천 차단된다.

사실 / 전달의 분리

고정(사실) — 각 슬라이드의 핵심 개념·정답 골격은 미리 확정된 “뼈대 노트”로 못 박는다. 절대 틀리면 안 되는 부분.
생성(전달) — 그 뼈대에 말투·비유·순서·개인화를 매번 새로 입힌다. 사실은 항상 정확하고, 전달은 매번 살아있다.

§3 · persona ON

배움의 세계

학생의 실제 학습 행동 — 슬라이드를 AI에 넣고 설명받기 — 에서 출발한다. 핵심은 설명이 지금 나를 위해 실시간으로 써지는 느낌이다. 학생은 이미 완성된 설명은 읽지 않는다.

Ch.3 문맥자유문법 · 슬라이드 12 / 40
SLIDE
CFG = (N, T, P, S)
  • N — 비단말 기호
  • T — 단말 기호
  • P — 생성 규칙
  • S — 시작 기호
🧑‍🏫 AI 교수 · 친근한 선배
이건 아까 3번 슬라이드에서 본 BNF랑 연결돼요. N은 ‘레고 설명서의 부품 이름’ 같은 거고
◀ 이전 🤔 아직 이해 안 돼요 ✅ 이해했어요, 다음 ▶

실시간 강의

타이핑 스트리밍으로 “지금 써지는” 몰입감. 앞 슬라이드를 기억하는 맥락 있는 설명. “안 돼요” 클릭은 약점 데이터로 축적.

QnA 튜터

슬라이드 내용에 바로 질문 → 실시간 응답. 소크라테스식 유도(되묻기) 가능. 넘어가기 전 미니 질문으로 이해 확인.

깊이 조절

“더 쉽게 / 더 깊게 / 예시 하나 더” 버튼으로 학생 수준에 맞춤.

페르소나 — 학생이 선택 배움의 세계 전용

페르소나는 말투·비유·순서·격려의 양·설명 깊이만 바꾼다. 개념의 정확성과 성적·채점 기준은 절대 못 바꾼다. 시험 출제·채점에는 전혀 관여하지 않는다. 처음엔 3개 정도로 시작.

페르소나스타일이런 학생에게
🎓 엄격한 노교수격식·정확, 대충 넘기면 다시 물음긴장감 있게 제대로
😊 친근한 선배편한 말투, 비유·드립부담 없이 편하게
⚡ 핵심만 콕군더더기 없이 요점만시험 직전 빠르게
🧩 소크라테스형답 대신 계속 되물음스스로 깊게 파는
🌱 다정한 격려형칭찬·천천히, 틀려도 괜찮다자신 없는 입문자
§4 · persona OFF

평가의 세계

공식 성적은 오직 오프라인 시험으로만. 학습 상호작용 데이터는 성적에 직접 쓰지 않아 정당성을 지킨다. 페르소나 없음, 모두에게 동일 기준.

출제

  • 슬라이드/뼈대 노트에 근거해서만 출제 — 안 배운 건 못 냄
  • 난이도·유형 배분은 사람이 지정
  • 검증 엔진으로 자체 검증 (정답이 유일한가?)
  • 최종 승인은 진짜 교수/조교
  • 학습 데이터 기반 약점 맞춤 출제 가능

채점 — 정당성 장치

  • 채점 전에 명시적 루브릭 확정
  • 일관성 검사 (같은 답 = 같은 점수)
  • 점수마다 근거 문장 부착
  • 사람 검토 루프 — 경계선·이상치만 검수
답안 유형예시채점 방식정당성
자동 검증형 파스트리, CFG로 문자열 생성, 문법 규칙 검증 엔진이 실제로 파싱해 정오 판정 기계적 · 100% 일관
서술·설계형 “언어 설계의 트레이드오프를 논하라” 루브릭 기반 LLM 채점 정당성 승부처
⭐ 검증 엔진 — 킬러 기능

학생이 낸 문법/파스트리를 실제로 파싱해서 정오를 판정하는 엔진. 출제 검증과 자동 채점 양쪽에서 핵심 역할을 하며, 이 과목이라서 가능한 차별점이다.

§5

비용 전략

“매번 진짜 생성”은 비싸다. 핵심 통찰 — “살아있는 느낌”과 “매번 실시간 생성”은 분리할 수 있다. 학생이 원하는 건 전자다. 그래서 미리 굽고, 필요할 때만 살아있게.

  1. 슬라이드 기본 설명은 한 번만 생성(또는 저렴한 모델)해 저장·캐시
  2. 화면에선 타이핑 애니메이션으로 스트리밍 → 매번 “지금 써지는” 느낌 (거의 공짜)
  3. 개인화(“아까 헷갈려하셨죠”)는 앞 한두 문장만 실시간 생성해 붙임
  4. 진짜 질문(QnA)이 들어올 때만 똑똑한 모델 투입
  5. 고정 부분(슬라이드·뼈대·시스템 프롬프트)은 프롬프트 캐싱

개인화 델타의 비율이 “살아있음 ↔ 비용”의 균형점을 결정한다 — 봐가며 적절히 조절.

§7

MVP 제안

“문법 문제 출제 → 학생 답안 자동 검증 → 피드백” 한 사이클. 이 하나가 “가르치고 → 평가하는 AI 교수”의 핵심을 증명한다.

  1. AI가 CFG 문제를 하나 낸다
  2. 학생이 파스 트리 / 문법 규칙으로 답한다
  3. 검증 엔진이 실제로 파싱해 정오를 판정한다
  4. AI가 “왜 틀렸는지” 개인 피드백을 준다
§8

미결 항목 · 다음 논의

보류하거나 아직 정하지 않은 것들. 다음 브레인스토밍은 여기서 이어간다.

  • 부정행위 방지 (오프라인 시험으로 상당 부분 해결, 세부는 보류)
  • 오프라인 답안 디지털화 — 종이+손글씨 OCR vs 태블릿 입력 (오독이 정당성 붕괴로 이어질 위험)
  • 이의제기(재채점) 흐름 상세
  • 개인화 델타 비율의 구체적 기준
  • 슬라이드 밖 질문 처리 방침
  • 여러 학생/반 관리, 진짜 교수와의 협업, 온보딩
  • 페르소나 최종 라인업 및 말투 프롬프트 설계
