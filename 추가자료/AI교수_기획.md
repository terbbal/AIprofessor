# AI 교수 프로젝트 기획서

> 프로그래밍 언어론을 가르치고 평가하는 AI 교수 — 브레인스토밍 정리
> 최종 수정: 2026-07-21

---

## 1. 프로젝트 개요

**한 줄 정의**: 슬라이드 기반으로 살아있는 강의를 하고, 오프라인 시험을 출제·채점하는 AI 교수.

| 항목 | 내용 |
|------|------|
| **과목** | 프로그래밍 언어론 (Programming Languages / 언어 생성 규칙) |
| **수준** | 대학 3학년 전공 |
| **형태** | 여러 학생 대상 강의 |
| **AI의 역할** | 가르치기 + 시험 출제 + 채점 |

**다루는 주제**: 형식 문법(BNF/EBNF, CFG), 파스 트리, 구문 분석, 의미론, 타입 시스템, 계산 모델(람다 계산법, 패러다임), 언어 설계.

### 프로젝트를 두 축으로 분리 (이번 MVP의 우선순위)

| 축 | 내용 | 성격 |
|-----|------|------|
| **가르치기 (Teaching)** | 슬라이드 실시간 설명 + QnA 튜터 + 페르소나 | **이번 MVP의 핵심** |
| **평가하기 (Assessment)** | 출제·채점, 검증 엔진 | 이번엔 **개념 증명만** |

> 두 축은 성격이 정반대다(아래 §2). 이번 사이클은 **가르치기를 제대로 만들고, 평가하기는 CFG 문제 1개 자동 채점으로 개념만 증명**하는 데 집중한다.

## 2. 핵심 설계 원칙

### 두 개의 분리된 세계

시스템은 성격이 정반대인 두 세계로 나뉘며, 이 분리가 아키텍처의 근간이다.

```
┌─────────────────────────────┐   ┌─────────────────────────────┐
│      🎭 배움의 세계            │   │      ⚖️ 평가의 세계            │
│      (페르소나 O)             │   │      (페르소나 X, 중립)        │
├─────────────────────────────┤   ├─────────────────────────────┤
│  • 슬라이드 강의              │   │  • 오프라인 시험 출제          │
│  • QnA 튜터                  │   │  • 채점 (자동검증 + 루브릭)     │
│  • 학생이 교수 스타일 선택     │   │  • 모두에게 동일 기준           │
│  분위기: 살아있는 선생님       │   │  분위기: 공정한 시험관          │
└─────────────────────────────┘   └─────────────────────────────┘
         친근함 · 개인화                  일관성 · 정당성
```

- **데이터는 한 방향으로만 흐른다**: 배움의 세계가 쌓은 약점 데이터 → 평가의 세계의 맞춤 출제에 활용 가능. 그 반대(채점이 페르소나 영향받기)는 원천 차단.

### 사실 / 전달의 분리

- **고정(사실)**: 각 슬라이드의 핵심 개념·정답 골격은 미리 확정된 "뼈대 노트"로 못 박는다. 절대 틀리면 안 되는 부분.
- **생성(전달)**: 그 뼈대에 말투·비유·순서·개인화를 매번 새로 입혀 전달한다.
- 효과: 사실은 항상 정확하고, 전달은 매번 살아있다.

## 3. 배움의 세계 (Teaching)

### 3.1 슬라이드 위의 실시간 강의

학생의 실제 학습 행동(슬라이드를 AI에 넣고 설명받기)에서 출발한 UX.

```
┌──────────────────────────────────────────────────────┐
│  Ch.3 문맥자유문법          슬라이드 12 / 40  [진도바 ▓▓▓░] │
├───────────────────────────────┬──────────────────────┤
│                               │  🧑‍🏫 AI 교수           │
│      [ PPT 슬라이드 이미지 ]     │  (실시간 타이핑 설명)   │
│   "CFG = (N, T, P, S)"        │  💬 질문하세요           │
├───────────────────────────────┴──────────────────────┤
│  [◀ 이전]   [🤔 아직 이해 안 돼요]   [✅ 이해했어요, 다음 ▶] │
└──────────────────────────────────────────────────────┘
```

**핵심 UX 원칙**: 학생은 "이미 완성된 설명"은 안 읽는다. 그래서 설명이 **지금 나를 위해 실시간으로 써지는 느낌**(타이핑 스트리밍)을 준다.

**똑똑해지는 지점**:
- 넘어가기 전 툭 던지는 미니 질문으로 진짜 이해했는지 확인 (버튼 연타 방지)
- 앞 슬라이드를 기억하는 맥락 있는 설명
- "안 돼요" 클릭이 약점 데이터로 축적
- "더 쉽게 / 더 깊게 / 예시 하나 더" 깊이 조절

### 3.2 QnA 튜터

- 슬라이드 내용에 대해 학생이 바로 질문 → 실시간 응답
- 소크라테스식 유도(정답을 바로 주지 않고 되묻기) 가능
- 슬라이드 밖 질문 처리 방침은 추후 결정 (자유 응답 vs "그건 N장에서" 유도)

### 3.3 페르소나 (학생이 선택)

**페르소나는 배움의 세계에만 투입된다. 시험 출제·채점에는 전혀 관여하지 않는다.**

| 페르소나 | 스타일 |
|----------|--------|
| 🎓 엄격한 노교수 | 격식·정확, 대충 넘기면 다시 물음 |
| 😊 친근한 선배 | 편한 말투, 비유·드립 |
| ⚡ 핵심만 콕 | 요점만 짧게 (시험 직전용) |
| 🧩 소크라테스형 | 답 대신 되물음 |
| 🌱 다정한 격려형 | 칭찬·천천히 (입문자용) |

- 언제든 변경 가능. 처음엔 **3개 정도**로 시작 권장.
- **바꿔도 되는 것**: 말투, 비유, 순서, 격려의 양, 설명 깊이.
- **절대 못 바꾸는 것**: 개념의 정확성, 성적·채점 기준.

## 4. 평가의 세계 (Assessment)

**공식 성적은 오직 오프라인 시험으로만.** 학습 상호작용 데이터는 성적에 직접 쓰지 않는다(정당성 확보). 페르소나 없음.

### 4.1 출제

- 강의 슬라이드/뼈대 노트에 **근거해서만** 출제 (출처 추적, 안 배운 건 못 냄)
- 난이도·유형 배분은 사람이 지정 (예: 쉬움4/중간4/어려움2, 서술형2)
- AI가 낸 문제를 검증 엔진으로 자체 검증 (정답이 존재하고 유일한가?)
- 최종 승인은 진짜 교수/조교
- 학습 데이터 기반 개인 맞춤 출제도 가능 (약점만 콕)

### 4.2 채점

| 유형 | 예시 | 채점 방식 | 정당성 |
|------|------|-----------|--------|
| **자동 검증형** | 파스트리, CFG 생성, 문법 규칙 | 검증 엔진이 실제로 파싱해 판정 | 기계적·100% 일관 ✅ |
| **서술·설계형** | "설계 트레이드오프를 논하라" | 루브릭 기반 LLM 채점 | 정당성 승부처 ⚠️ |

**서술형 채점 정당성 장치**:
- 채점 **전에** 명시적 루브릭 확정 (항목별 배점)
- 일관성 검사 (같은 답 여러 번 채점, 비슷한 답 점수 차이 탐지)
- 점수마다 근거 문장 부착 (이의제기 대응 근거)
- 사람 검토 루프 (교수는 경계선·이상치만 검수, 0에서 채점 X)

### 4.3 검증 엔진 ⭐ (킬러 기능)

학생이 낸 문법/파스트리를 **실제로 파싱해서** 정오를 판정하는 엔진. 출제 검증과 자동 채점 양쪽에서 핵심 역할. 이 과목이라서 가능한 차별점.

## 5. 비용 전략

"매번 진짜 생성"은 비싸므로, **미리 굽고 필요할 때만 살아있게**:

1. 슬라이드 기본 설명은 한 번만 생성(또는 저렴한 모델)해 저장·캐시
2. 화면에선 타이핑 애니메이션으로 스트리밍 → 매번 "지금 써지는" 느낌 (거의 공짜)
3. 개인화("아까 헷갈려하셨죠")는 앞 한두 문장만 실시간 생성해 붙임
4. 진짜 질문(QnA)이 들어올 때만 똑똑한 모델 투입
5. 고정 부분(슬라이드·뼈대·시스템 프롬프트)은 프롬프트 캐싱

> "살아있는 느낌"과 "매번 실시간 생성"은 분리 가능하다. 학생이 원하는 건 전자다.
> 개인화 델타의 비율은 "살아있음 ↔ 비용" 균형을 봐가며 적절히 조절.

## 7. MVP 제안

> **"문법 문제 출제 → 학생 답안 자동 검증 → 피드백" 한 사이클**
>
> 1. AI가 CFG 문제를 하나 낸다
> 2. 학생이 파스 트리/문법 규칙으로 답한다
> 3. 검증 엔진이 실제로 파싱해 정오 판정
> 4. AI가 "왜 틀렸는지" 개인 피드백 제공

이 한 사이클이 "가르치고 → 평가하는 AI 교수"의 핵심을 증명한다.
