# 🧑‍🏫 AI 교수

## 슬라이드를 보고 실시간으로 가르치는 AI 교수

프로그래밍 언어론 · 프로토타입 데모

<br>

> "이미 완성된 설명"은 안 읽는다.
> 그래서 **지금 나를 위해 실시간으로 써지는** 설명을 준다.

---

## 무엇을 푸는가

- 학생은 이미 **슬라이드를 AI에 넣고 설명받으며** 공부한다.
- 그 학습 행동을 그대로 **하나의 강의 경험**으로 만든다.
- 교수가 올린 자료 위에서, AI가 **페이지 단위로 설명하고 질문을 받는다.**

**한 줄 정의**
> 슬라이드 기반으로 살아있는 강의를 하고, (다음 단계로) 시험을 출제·채점하는 AI 교수

---

## 두 개의 분리된 세계

| 🎭 배움의 세계 | ⚖️ 평가의 세계 |
|----------------|----------------|
| 페르소나 O (개인화) | 페르소나 X (중립) |
| 슬라이드 강의 · QnA 튜터 | 출제 · 채점 · 검증엔진 |
| 친근함 · 개인화 | 일관성 · 정당성 |

- 데이터는 **한 방향**으로만 흐른다: 배움 → 평가(맞춤 출제).
- 채점이 페르소나에 영향받는 일은 **원천 차단**.
- **오늘 데모 = 배움의 세계** / 평가는 다음 단계.

---

## 핵심 설계 원칙 — 사실 / 전달의 분리

- **고정(사실)**: 슬라이드(PDF) 페이지의 실제 내용 = 못 박힌 근거.
  - AI는 이 근거를 **벗어난 개념을 지어내지 못한다.**
- **생성(전달)**: 그 위에 말투·비유·순서·개인화를 매번 새로 입힌다.

➡️ **사실은 항상 정확하고, 전달은 매번 살아있다.**

---

## 오늘 데모 범위

- ✅ **로그인** → 교수 / 학생 역할 분리
- ✅ **교수**: 수업 자료(PDF·이미지) 업로드
- ✅ **학생**: 학습 화면 = 왼쪽 PDF 한 페이지 + 오른쪽 설명 챗봇
- ✅ 페이지를 넘기면 **그 페이지 내용 기반**으로 설명 재생성
- ✅ **교수 페르소나 3종** 선택으로 말투 전환
- ✅ 오른쪽 챗봇에 **자유 질문**

---

## 데모 흐름

1. **로그인 화면** — 학생 / 교수 선택
2. (교수) **수업 자료 올리기** → PDF 업로드
3. (학생) **학습 시작** → 학습 화면 진입
4. **왼쪽**: PDF 한 페이지씩 렌더
5. **오른쪽**: AI 교수가 그 페이지를 실시간 설명(타이핑)
6. **이전/다음**으로 페이지 이동 → 설명도 따라 갱신
7. **페르소나 전환** · **질문** 시연

---

## 시스템 구조

```
[브라우저]
  로그인 → 교수 화면 / 학생 화면
                     │
        학습 화면 (2단 레이아웃)
   ┌────────────┬─────────────────┐
   │  PDF 뷰어   │   설명 챗봇       │
   │ (pdf.js)   │  (스트리밍)       │
   │  페이지     │                  │
   │  텍스트 추출 ─────► /api/tutor  │
   └────────────┴─────────┬────────┘
                          ▼
               Claude (서버 라우트에서만 호출)
```

- 페이지 텍스트를 **근거**로 넘겨 → 그 내용만으로 설명/답변.

## 기술 스택

| 계층 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 14** | 프론트+백엔드 한 몸, API 라우트 |
| PDF 렌더 | **pdf.js** | 브라우저에서 페이지별 렌더·텍스트 추출 |
| LLM | **@anthropic-ai/sdk** | 스트리밍(타이핑 효과) |
| 스타일 | **Tailwind CSS** | 빠른 미니멀 UI |
| 배포(예정) | **Vercel** | git push → 자동 배포 |

## 모델 전략 — 작업별 등급 분리

| 작업 | 모델 | 언제 |
|------|------|------|
| 슬라이드 설명 | **Haiku 4.5** | 저렴·빠르게 |
| QnA 튜터 | **Sonnet 5** | 실제 질문할 때 |
| 서술형 채점(다음) | **Opus 4.8** | 정당성 필요한 채점 |

**비용 전략**: 프롬프트 캐싱 + 스트리밍 재생 → 학생이 늘어도 비용이 선형 폭증하지 않음.

---

## 보안 원칙

- **API 키는 서버(API 라우트)에서만** 사용.
- 브라우저 코드 / `NEXT_PUBLIC_*` 에 **절대** 노출하지 않음.
- 모든 Claude 호출은 `app/api/*/route.ts` 를 거친다.

> 지금은 키 없이도 **데모 모드**(페이지 내용 기반 한국어 안내)로 동작.
